Control Git

Gestiona versiones como pro.

1. Iniciar
git init inicializa un nuevo repositorio en tu carpeta local, creando la carpeta oculta .git.
2. Estado
git status es vital. Muestra qué archivos han cambiado, cuáles están preparados y cuáles no.
3. Preparar
git add archivo mueve cambios al "staging area", preparándolos para ser guardados oficialmente.
4. Guardar
git commit -m "mensaje" toma una "foto" del código actual. El mensaje debe ser descriptivo.
5. Descargar
git pull descarga cambios del repositorio remoto y trata de fusionarlos con tu versión local.
6. Subir
git push sube tus commits locales a la nube (GitHub/GitLab) para que otros los vean.
7. Crear Rama
git branch nombre crea una línea temporal de desarrollo para trabajar sin romper el código principal.
8. Cambiar Rama
git checkout nombre te mueve a otra rama. En versiones nuevas se usa git switch.
9. Fusionar
git merge rama une los cambios de una rama secundaria a la rama en la que estás actualmente.
10. Remoto
Un repositorio alojado en internet (ej: GitHub). Se conecta con git remote add origin url.
